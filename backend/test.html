Enhanced Text Processing Features

This page demonstrates the enhanced text processing features available in the Fake News Detection system.

1. Language Detection

The system can automatically identify the language of input text.

Example: "This is a sample text in English language for testing purposes." 
{
  "language_code": "en",
  "language_name": "English",
  "confidence": 0.95,
  "supported": true
}

2. Entity Extraction

Identifies named entities such as people, organizations, and locations.

Example: "The President of the United States, Joe Biden, visited Berlin, Germany last week." 
{
  "entities": {
    "PERSON": 1,
    "GPE": 2,
    "DATE": 1,
    "ORG": 1
  },
  "entity_count": 5
}

3. Readability Metrics

Calculates standardized readability scores to assess text complexity.

Example: "The analysis of various textual features can reveal patterns associated with fake news." 
{
  "flesch_reading_ease": 42.31,
  "flesch_kincaid_grade": 12.84,
  "gunning_fog": 14.53,
  "smog_index": 10.62,
  "coleman_liau_index": 12.57,
  "automated_readability_index": 13.92,
  "average_grade_level": 12.9
}

4. Text Uniqueness Analysis

Measures lexical diversity and content uniqueness.

Example: "The quick brown fox jumps over the lazy dog." 
{
  "unique_words_ratio": 1.0,
  "lexical_diversity": 1.0,
  "content_hash": "9e107d9d372bb6826bd81d3542a419d6"
}

5. Propaganda Technique Detection

Identifies common propaganda techniques in text.

Example: "SHOCKING NEWS! The radical left doesn't want you to know this INCREDIBLE truth." 
{
  "techniques": {
    "name_calling": 1,
    "exaggeration": 2,
    "card_stacking": 1
  },
  "propaganda_score": 28.57
}

How to Use the API

To access these features, use the following API endpoints:

  • /analyze/enhanced - Full analysis with enhanced features
  • /detect-language - Language detection only
  • /analyze/comprehensive - Comprehensive text analysis

For full API documentation, visit http://localhost:8000/docs when the server is running.
